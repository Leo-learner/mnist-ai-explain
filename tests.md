# 测试说明

本文档用于课程报告中的测试分析部分，覆盖功能测试、性能测试和稳定性测试。

## 1. 功能测试

| 测试项 | 测试方法 | 预期结果 |
| --- | --- | --- |
| 模型训练 | 运行 `python train.py --epochs 3` | 自动下载 MNIST，完成训练，并生成 `models/mnist_cnn.pth` |
| 单图推理 | 运行 `python predict.py --image path/to/digit.png` | 终端输出预测数字、置信度和 10 类概率 |
| 页面启动 | 运行 `./run.sh` 或 `run.bat` | 浏览器打开 Streamlit 页面 |
| 图片上传 | 上传 PNG、JPG、JPEG 或 BMP 手写数字图片 | 页面展示原图、裁剪居中后的 28×28 预处理图、预测结果和概率柱状图 |
| 大留白图片 | 上传数字较小、四周留白较多的图片 | 系统自动裁剪有效笔画区域，降低数字被压缩过小导致的误识别 |
| 离线解释 | 不配置 `API_KEY` 后上传图片 | 系统使用本地模板输出四段式解释 |
| 在线解释 | 配置 `API_KEY`、`LLM_MODEL`、`LLM_API_BASE` 后上传图片 | 系统调用兼容接口生成大模型解释，失败时自动回退本地模板 |

## 2. 性能测试

| 测试项 | 测试方法 | 记录指标 |
| --- | --- | --- |
| 训练耗时 | 记录 `python train.py --epochs 3` 的开始和结束时间 | 总训练时间、测试集准确率 |
| 推理速度 | 连续上传多张手写数字图片 | 页面响应是否流畅，单张图片推理是否在可接受时间内完成 |
| 资源占用 | 在 CPU 或普通笔记本环境下运行系统 | 是否能正常启动，是否出现明显卡顿 |

建议在报告中记录训练结束输出的 `Test Accuracy`。本项目在当前电脑完整训练 3 轮后，测试集准确率约为 `99%`，实际结果会因设备和随机因素略有变化。

## 3. 稳定性测试

| 测试项 | 测试方法 | 预期结果 |
| --- | --- | --- |
| 未上传图片 | 打开页面但不上传文件 | 页面提示用户先上传图片 |
| 错误文件 | 上传被破坏或伪装成图片的文件 | 页面给出友好错误提示，不崩溃 |
| 空白图片 | 上传几乎没有数字笔画的空白图片 | 页面提示未检测到明显数字笔画 |
| 模型缺失 | 临时移走 `models/mnist_cnn.pth` 后启动页面 | 页面提示先训练模型 |
| 多次上传 | 连续上传不同图片 | 页面每次刷新预测结果和概率图 |
| 无 API Key | 不配置任何大模型密钥 | 系统仍可离线运行，并输出本地解释 |

## 4. 报告撰写建议

- 在功能测试中截图展示：原图、预处理图、预测数字、置信度、概率柱状图和解释文本。
- 在性能测试中记录：训练轮数、批大小、学习率、测试集准确率和训练耗时。
- 在稳定性测试中说明：系统对未上传、错误图片、模型缺失、无 API Key 等情况均有提示或回退机制。
